## Qué son estos recursos

Las **checklists y ejemplos oficiales de AESIA** sirven como apoyo práctico para autoevaluaciones, revisiones internas y preparación de evidencias de cumplimiento en torno al Reglamento de IA.

## Qué incluyen

Estas plantillas están pensadas para convertir requisitos normativos en preguntas, controles y evidencias revisables.

### 1. Gestión de riesgos y calidad

- **Checklist de gestión de riesgos** *(archivo oficial: `Gestión de riesgos_Checklist.xlsx`)* — Revisión del artículo 9 sobre sistema de gestión de riesgos.
- **Ejemplo ilustrativo de desarrollo de un sistema de gestión de riesgos** *(archivo oficial: `05.01 Ejemplo ilustrativo del desarrollo de un sistema de gestión de riesgos.xlsx`)* — Caso práctico de evaluación de riesgos paso a paso.
- **Checklist de gestión de calidad** *(archivo oficial: `Gestión de Calidad_Checklist.xlsx`)* — Auditoría del sistema de gestión de la calidad exigido por el artículo 17.

### 2. Requisitos técnicos principales

- **Checklist de gobernanza del dato** *(archivo oficial: `Gobernanza del dato_Checklist.xlsx`)* — Validación de conjuntos de datos, sesgos, procedencia y representatividad.
- **Checklist de transparencia** *(archivo oficial: `Transparencia_Checklist.xlsx`)* — Revisión de instrucciones de uso e información obligatoria para implantadores.
- **Checklist de supervisión humana** *(archivo oficial: `Supervisión Humana_Checklist.xlsx`)* — Verificación de mecanismos de control y supervisión humana.
- **Checklist de documentación técnica** *(archivo oficial: `Documentación tecnica_Checklist.xlsx`)* — Apoyo para expediente técnico, artículo 11 y Anexo IV.

### 3. Seguridad y robustez técnica

- **Checklist de precisión** *(archivo oficial: `Precision_Checklist.xlsx`)* — Revisión de métricas de precisión declaradas para el sistema.
- **Checklist de solidez** *(archivo oficial: `Solidez_Checklist.xlsx`)* — Validación de resistencia ante fallos, perturbaciones y entradas anómalas.
- **Checklist de ciberseguridad** *(archivo oficial: `Ciberseguridad_Checklist.xlsx`)* — Revisión de defensas ante ataques, manipulación o envenenamiento.

### 4. Trazabilidad y operación

- **Checklist de registros** *(archivo oficial: `Registros_Checklist.xlsx`)* — Revisión del logging, trazas y conservación de registros.
- **Checklist de vigilancia poscomercialización** *(archivo oficial: `Vigilancia Poscomercializacion_Checklist.xlsx`)* — Apoyo para monitorización en producción y seguimiento posterior.
- **Checklist de gestión de incidentes** *(archivo oficial: `Gestión de incidentes_Checklist.xlsx`)* — Protocolo para reporte, análisis e investigación de incidentes graves.

## Cómo utilizarlas bien

1. **Empieza por el manual** para entender el alcance de cada lista de control.
2. **Relaciona cada checklist** con la guía técnica correspondiente dentro de este portal.
3. **Úsalas como rúbrica interna**, no como sustituto de criterio jurídico o técnico propio.
4. **Acompaña cada respuesta con evidencia**: políticas, procedimientos, métricas, registros o documentación técnica.

## Sugerencia práctica

Si estás montando un flujo de cumplimiento, una buena secuencia suele ser:

1. **Clasificación y contexto** → guía introductoria + guía práctica.
2. **Base de cumplimiento** → evaluación de conformidad, riesgos y calidad.
3. **Controles técnicos** → datos, transparencia, supervisión humana, precisión, solidez y ciberseguridad.
4. **Operación y trazabilidad** → registros, vigilancia poscomercialización e incidentes.
