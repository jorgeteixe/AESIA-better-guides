This MVP reorganises the **16 AESIA supervision guides** into a lightweight docs experience built with **Next.js + Fumadocs**.

## What is here

- A readable entrypoint to the AESIA guide corpus
- A cross-guide deep summary
- Individual guide pages derived from the markdown transcriptions
- Official references and source links
- A practical checklist/resources section

## Why this exists

The original materials are valuable, but they are hard to navigate as a pile of PDFs. This project is a first pass at turning them into:

- **readable docs**
- **searchable knowledge**
- **compliance playbooks**
- **future agent/RAG-ready content**

## Current scope

This is intentionally simple:

- no heavy redesign yet
- no advanced search tuning yet
- no custom workflows yet
- no automated article-to-obligation extraction yet

The goal is just to stand up a clean, usable **MVP**.

## Suggested next steps

- add happy paths by role (SME, provider, deployer, auditor)
- add obligation → evidence matrices
- add examples and template artifacts
- add AI-assisted Q&A grounded on the guides
- add versioning/diff tracking when AESIA updates the source materials
